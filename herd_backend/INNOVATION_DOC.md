# 中药毒性预测 — 创新点文档

## 创新层次总览

```
主创新: 化合物→草药跨粒度知识迁移框架
  ├── 化合物级预训练 (CompoundToxModel)
  ├── 多头注意力聚合 (CompoundAttentionAggregator)
  └── 上下文门控注入 (GatedPriorProtoNet)

次级创新: ECC-Adaptive+ 自适应链式多标签预测
  ├── MI-based 链排序
  └── 双轮集成 (P1 + P2)

辅助技术: 知识图谱正则化 + 数据增强策略
```

## 一、主创新：化合物→草药跨粒度知识迁移框架

### 1.1 核心问题

草药毒性预测的根本困难是**粒度不匹配**：已知的毒性知识主要在分子层面（化合物），而预测目标在草药层面（多化合物混合物）。传统方法要么忽略分子信息，只用草药级手工特征；要么简单平均所有化合物的毒性信号，无法区分关键成分和无关成分。

### 1.2 框架设计

```
分子层面                      草药层面
─────────                    ─────────
Morgan FP [1024]              
    ↓                        
CompoundToxModel              
    ↓                        
化合物毒性概率 [n_c, 5]       
    ↓                        
CompoundAttentionAggregator   
  (学习: 哪个化合物对哪类     ──→  草药先验 [5]
   毒性最重要？)                 
                                  ↓
                            GatedPriorProtoNet
                              (门控: 信任这个先验
                               多少？)
                                  ↓
                            最终毒性预测 [5]
```

**关键设计选择**：
- **Morgan FP 作为跨粒度的通用语言**：无论西药化合物还是中药化合物，都用同一套 1024 维分子指纹表示
- **两阶段训练**：化合物模型和草药模型分开训练，避免循环依赖
- **门控注入而非替换**：prior 以残差形式注入，最坏情况 gate→0 → 退化为不使用 prior 的基线

### 1.3 CompoundAttentionAggregator

**为什么不用 Mean Pool？**

以黄连为例：含小檗碱(抗菌)、黄连碱(抗炎)、巴马汀(抗菌)。对肝毒性而言，小檗碱是已知的 CYP3A4 抑制剂，黄连碱贡献微弱。Mean pool 将三者等权平均，把关键信号稀释了 3 倍。

**多头注意力设计**：

每个毒性标签有独立的注意力头——肝毒性关注代谢酶抑制剂，肾毒性关注转运体底物，二者关注的化合物集合不同。

```python
attn_w = softmax(Linear(Tanh(Linear(probs, 5→32)), 32→5))  # [n_c, 5]
aggregated = (attn_w * probs).sum(dim=0)  # 加权和 [5]
```

**消融证据**：
| 配置 | AUC | Macro-F1 |
|------|-----|----------|
| Full Model | 0.6853 | 0.6801 |
| - Knowledge Transfer (no prior) | 0.6768 (-0.9pp) | 0.6724 |
| - Aggregator (mean pool) | 0.6803 (-0.5pp) | 0.6736 |

- 去掉整个知识迁移链：AUC -0.9pp → 分子层信息确实在帮助草药层预测
- 用 mean pool 替代注意力：AUC -0.5pp → 聪明的聚合比简单平均好

### 1.4 GatedPriorProtoNet

**设计**：`h = encoder(x) + gate(prior) * prior_proj(prior)`

**两个关键细节**：

1. **bias=0 初始化**：`nn.init.zeros_(gate[-2].bias)` → sigmoid(0)=0.5。训练初期模型"信任一半"先验，随后梯度决定开关方向。

2. **Gate 只看 prior，不看 features（有意的简化）**：
   - 我们测试了 context-aware gate `gate(cat(h, prior))`（Linear 261→128），在 252 样本上因 33K 额外参数过拟合，AUC -0.4pp
   - 当前设计 `gate(prior)` 仅 100 参数，在小数据上更稳定
   - 论文中写为 **design choice**：小数据场景下，参数效率优先于表达能力

### 1.5 可解释性：注意力权重可视化

多头注意力权重 `attn_w [n_compounds, 5]` 可直接解释。以黄连为例：

| 化合物 | 肝毒性 | 肾毒性 | 心脏毒性 |
|--------|--------|--------|----------|
| 小檗碱 | **0.62** | 0.15 | 0.08 |
| 黄连碱 | 0.23 | 0.18 | 0.12 |
| 巴马汀 | 0.15 | **0.67** | 0.05 |

→ 论文可展示："模型学到了小檗碱是黄连肝毒性的主因，巴马汀是肾毒性的主因"

## 二、次级创新：ECC-Adaptive+ 自适应链式预测

### 2.1 核心问题

多标签毒性之间存在依赖关系。肝损伤常导致肾损伤（肝肾综合征），心脏毒性和神经毒性可能共享离子通道机制。独立预测每个标签会忽略这些依赖。

### 2.2 链式预测 + MI 排序

```
链顺序 (MI贪心):
  MI(i,j) = mutual_info(label_i, label_j)  # 标签间依赖强度
  贪心构造: 每步选与已选标签 MI 总和最大的未选标签

消融发现:
  MI × standalone_AUC = MI Only (链顺序完全一致)
  → standalone_aucs（嵌套3-fold CV）不改变排序，可安全移除
```

**消融证据**：
| 链策略 | AUC |
|--------|-----|
| MI 贪心 (当前) | 0.6853 |
| Random | 0.6798 (-0.6pp) |
| Fixed Forward | 0.6823 (-0.3pp) |
| Fixed Reverse | 0.6824 (-0.3pp) |

链顺序影响（~0.3-0.6pp）远小于数据增强（~4.3pp），说明 ECC 的真正价值不在排序策略，而在利用标签依赖进行条件预测。

### 2.3 双轮集成 (P1 + P2)

```
P1: 训练条件=真实标签 → 推理条件=val_probs (软)
P2: 训练条件=软化标签(0.8*l+0.1) → 推理条件=P1预测(软)

最终: (P1 + P2) / 2
```

**P2 的设计逻辑**：P1 推理用的是软概率，P2 训练时用软化标签模拟软条件，使训练-推理分布更一致。这是**隐式计划采样**——通过软化训练条件降低曝光偏差。

**消融证据**：去掉 P2 → Macro-F1 -0.9pp (AUC 仅 -0.4pp)。P2 的主要价值在提升分类决策质量（F1），而非排序质量（AUC）。

## 三、辅助技术（非独立创新）

### 3.1 知识图谱正则化

Jaccard 化合物共现相似度作为嵌入正则。**消融显示重要**（AUC -2.4pp），但方法本身是图学习的标准技术，**不作为独立创新点**。

### 3.2 数据增强组合

`Label Smoothing(0.95/0.025) → DropAdd(0.2/0.01) → Mixup(α=0.2)`

**顺序重要**：先平滑硬标签，再 mixup。反向操作会减弱平滑效果。

**消融显示关键**（Mixup AUC -4.3pp, DropAdd AUC -1.8pp），但方法本身是通用技术，放在 Implementation Details。

## 四、探索过但放弃的方向

| 方向 | 尝试 | 结果 | 原因 |
|------|------|------|------|
| Context-aware gate | gate(cat(h, prior)) | AUC -0.4pp | 33K 参数在小数据过拟合 |
| 相似度矩阵插值 | mixup 同步插值知识正则目标 | F1 -0.9pp | 插值稀释了相似度信号 |
| ECC 训练软化 | 训练条件用软标签 | F1 -0.6pp | 破坏硬边界判别 |
| 推理二值化 | 推理时 val_probs > 0.5 | F1 -1.6pp | 丢失概率细粒度信息 |
| P1/P2 不等权 | 0.4/0.6 动态加权 | AUC -0.2pp | 固定权重过拟合验证集 |

## 七、当前局限与下一步

1. **数据规模** (252 草药)：限制了深度模型和 context-aware gate 的表达空间。外部数据（UniTox ~200K 西药）是最有潜力的突破口。
2. **分子表示**：仅用 Morgan FP，未利用分子图结构（GNN）。
3. **Gate 设计**：当前 gate 只看 prior，论文中需明确标记为"小数据下的设计选择"。
4. **链顺序**：MI 贪心比随机链仅好 0.6pp，链排序不是当前瓶颈。
